# 実況ツイートメーカー（Live Tweeting Maker）開発計画

## 背景・目的
動画のリアルタイム視聴（リアタイ）を逃したユーザーが、後から録画やアーカイブを視聴する際に、動画の進行と連動したタイムスタンプ付きの実況コメントを記録するためのブラウザ完結型Webアプリケーションを開発する。

## 目標
1.  **ブラウザ完結**: サーバーレスで運用コストをゼロにする。データはブラウザ（Local Storage）に保存し、ファイルとしてエクスポート/インポート可能にする。
2.  **正確なタイムスタンプ**: 動画の再生開始に合わせるための「カウントダウン機能」と、再生中のズレを補正する「微調整（＋秒／－秒）ボタン」を提供する。
3.  **使いやすいUI**: 実況用途に特化し、常に入力欄が表示され、X（Twitter）の制限を意識した文字数カウント機能を提供する。

## 技術スタック
*   **フロントエンド**: React (UI構築), Vite (ビルドツール), TypeScript (型安全)
*   **スタイリング**: Tailwind CSS (ユーティリティファーストCSS)
*   **アイコン**: Lucide React
*   **データ保存**: `localStorage` (ブラウザ内保存)
*   **ファイル入出力**: ブラウザのFile API (エクスポート: `Blob` / インポート: `FileReader`)
*   **デプロイ想定**: GitHub Pages 等の静的ホスティング

## コア機能と要件

### 1. タイマー同期機能
*   **カウントダウン・スタート**: 動画の再生ボタンを押すタイミングを合わせるため、例えば「3・2・1・スタート」といったカウントダウン後にタイマーを開始する。
*   **タイマー表示**: `HH:MM:SS` 形式で現在の経過時間を表示する。
*   **微調整機能**: 動画とタイマーがズレた場合のために、「+1秒」「-1秒」「+10秒」「-10秒」などのオフセット調整ボタンを設ける。

### 2. 実況投稿機能
*   **常時表示の入力エリア**: 画面をスクロールしても、テキスト入力フォームが常に画面下部（または固定位置）に表示され、いつでも書き込めるようにする。
*   **文字数カウント**: 入力中のテキストの文字数をリアルタイムで表示し、X(Twitter)の制限（全角140文字/半角280文字相当）の目安とする。
*   **投稿アクション**: Enterキー（または送信ボタン）で、その瞬間の「タイマー時間」と「テキスト」を記録し、タイムラインに追加する。

### 3. データ管理機能（インポート/エクスポート）
*   **ローカル保存**: リロードや誤ってタブを閉じてもデータが消えないよう、投稿ごとに `localStorage` に自動保存する。
*   **エクスポート**: 記録した実況データを、テキストファイル（CSVやJSON、またはプレーンテキスト形式）としてダウンロード（保存）する。
*   **インポート**: 過去にダウンロードしたファイルを読み込み、タイムラインを復元して続きから実況できるようにする。

## UIレイアウトのイメージ（ワイヤーフレーム案）

*   **ヘッダー**: アプリ名、エクスポート/インポートボタン
*   **上部**: 
    *   メインタイマー表示（大きく見やすく）
    *   タイマー操作パネル（スタート/ストップ、カウントダウン設定）
    *   微調整パネル（-10s, -1s, +1s, +10s）
*   **中央部（スクロール領域）**: 
    *   タイムライン表示（新しい投稿が下に追加されていく、または新しい投稿が上に来る形式）
    *   各投稿には「タイムスタンプ」「テキスト」「削除/編集ボタン」を配置。
*   **下部（固定表示）**:
    *   テキスト入力エリア（複数行対応）
    *   文字数カウンター
    *   送信ボタン

## 開発ステップ

1.  **プロジェクトの初期化**: `npm create vite@latest` を使用して React + TS プロジェクトをセットアップ。Tailwind CSS の導入。
2.  **UIのモックアップ**: タイマー部分、タイムライン、固定入力フォームのガワ（見た目）を作成。
3.  **タイマー機能の実装**: カウントダウン機能、時間計測ロジック、オフセット調整機能を持つカスタムフック（`useTimer`）の作成。
4.  **実況機能の実装**: 入力フォームとタイムラインの連携、文字数カウント機能の実装。
5.  **データ永続化とファイル入出力の実装**: `localStorage` との同期処理、CSV/JSON のエクスポート・インポート機能の実装。
6.  **ブラッシュアップ**: スクロール挙動の調整、キーボードショートカット（Enterで送信など）の整備、全体的なスタイリングの調整。

## テスト・検証項目
*   カウントダウン後、タイマーが正確に開始されるか。
*   微調整ボタンを押した際、現在時刻の表示と、その後に記録されるタイムスタンプに正しく反映されるか。
*   文字数カウントが正しく機能するか（改行や特殊文字の扱い）。
*   エクスポートしたファイルを再度インポートした際、データが完全に復元されるか。
*   ページをリロードしても、直前の状態（タイマーはストップしている想定）とタイムラインが維持されるか。
